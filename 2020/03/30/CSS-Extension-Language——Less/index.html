Less

Abstract

Less (which stands for Leaner Style Sheets) is a backwards-compatible language extension for CSS. There are a variety of features in Less including Variables,Nesting,Mixins,Functions and so on.Less could improve our efficiency while coding in a front end program and make it easier for us to maintain our programs. This article would give a quick overview of these features.

Comments

Both block-style and inline comments may be used in Less

Importing

Just like the importing rule of CSS,we can import a .less files or .css files in any .less files. The extension is optionally optionally soecified for .less files.

Variables

In a less file, we can easily use variables. The name of variable starts with @.

Nesting

Less mimics the structure of HTML . It gives you the ability to use nesting instead of only cascading.

Nested At-Rules and Bubbling

At-rules such as @media or @supports can be nested in the same way as selectors. The at-rule is placed on top and relative order against other elements inside the same ruleset remains unchanged. This is called bubbling.

Mixins

This feature provides a method of including a bunch of properties from one rule-set into another rule-set in a .less file. We first defined a rule-set . Then,we just have to drop in the name of the css where we want the properties.

Operations

Arithmetical operation + - * /could operate on any number, color or variable.

Escaping

Escaping allows you to use any arbitrary string as property or variable value. Anything inside ~"anything" or ~'anything' is used as is with no changes except interpolation.

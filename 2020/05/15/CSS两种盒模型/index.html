CSS两种盒模型

概述

CSS有两种盒模型分别为:标准W3C盒模型和IE盒模型，我们可通过css3新增的属性 box-sizing: content-box | border-box分别设置盒模型为标准模型（content-box）和IE模型（border-box） 。

CSS盒模型

所有的HTML元素都可以看成是盒模型(box model),而这个盒子主要由四个部分组成:

属性 内容
margin 外边距，盒子与盒子之间的距离
border 边框，盒子边框
padding 内边距，盒子里面的内容和盒子边框的填充区域
content 内容，盒子中存放文本或图片的区域

标准模型

在CSS中，我们可以通过box-sizing:content-box；将盒子设置为标准模型（盒子默认为标准模型）。

标准模型的特征是: widt = content ,即盒子的宽度等于盒子的内容且不含内边距和边框，当我们改变标准模型的内边距甚至边框的数值的时候会使得盒子实际占有的空间变大。

它在页面中实际占有的宽高为：(50px * 2)(border) + (50px * 2)(padding) + 100px(content) = 300px

IE模型(怪异盒模型)

在CSS中，我们可以通过box-sizing:border-box；将盒子设置为 IE 模型（也叫做怪异盒子） 。

IE模型的特征是: width = content + padding + border，即盒子的宽度等于盒子的内容加上内边距和边框，当我们改变标准模型的内容宽高,内边距甚至边框的数值的时候不会使得盒子实际占有的空间发生改变。

它在页面中实际占有的宽高为：100px，但可以看到，由于 border + padding 已经等于 100px 了，所以这里 content 已经被压缩到只剩 0 了，实际盒子看起来已经无法很好的存放 content 内容了

（tip：为了防止盒子在页面中实际占有面积超出预计以出现横向滚动条，通常会有开发者将最外部的盒子设为 IE 模型 ）
